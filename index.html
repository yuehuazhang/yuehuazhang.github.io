FreeCAD中类型的动态创建

发表于 2022-01-05 更新于 2022-01-06 分类于 FreeCAD

  在我们读取数据时，会根据保存的数据的内容创建对应的类，从而正确的读取数据，根据保存数据创建类就是一种类的动态创建。本博客下面主要讲述FreeCAD中类的动态创建过程。

实现动态创建的重要材料

  在FreeCAD的type.cpp文件中，和动态创建非常相关的几个变量或者说类我们先介绍一下：
Type 类：这个类中封装了类的一般操作，比如创建实例，查找其父类等，这个类会和一个具体的类绑定，通过该类创建出对应的类，这个类应该是动态创建的接口类。
TypeData类： 是保存对应的绑定类的具体数据的类，Type 类应该说是对TypeData类操作的一个接口，TypeData中包含一个Type 类型的变量。
Type类中有两个非常重要的静态变量：
map Type::typemap;
vector Type::typedata;
这两个变量最终保存了注册类的名称以及创建类的函数。
  FreeCAD所有的类型都是继承自BaseClass类，这个类中有一个静态的Type即classTypeId（子类中会重写覆盖），正是通过这个，所有的类型和Type绑定，实现的动态创建的相关操作。
  所有继承自BaseClass类的类都有一个宏 TYPESYSTEM_HEADER();该宏重新声明了以下函数：
static Base::Type getClassTypeId(void);
static void init(void);
static void *create(void);
以及一个变量
static Base::Type classTypeId；
这些变量在基类中就存在，重新声明会覆盖掉以前的函数以及变量。后面我们会看到这几个函数和变量在该类型的动态创建很有关系。在类的实现中我们可以看到这个宏的定义类似如下所示：
TYPESYSTEM_SOURCE(Base::AbortException,Base::Exception)
这个是TYPESYSTEM_HEADER宏声明的实现。
有了以上宏定义的内容，我们就可以实现动态创建了。

动态创建的实现

  动态创建的一个基本的思路是这样的：针对一个类首先得有一个注册，注册的目的是将一个类名和类型实例创建的函数（static）绑定，同时保存在一个容器里面。下面是具体的创建方式。
  我们注册一个变量的时候会调用其init函数这个函数的实现在TYPESYSTEM_SOURCE中，这个函数主要进行了以下的操作：
  创建了一个TypeData，这个TypeData将类名、该类的父类的Type以及创建方法进行了保存。同时将这个创建的TypeData放入Type::typedata中。另外将类名以及在Type::typedata中的索引保存到Type::typemap中。

用VS2019编译FreeCAD

发表于 2021-08-15 更新于 2022-01-17 分类于 FreeCAD

   本博客主要讲怎样在VS2019中编译FreeCAD。VS2019集成可cmake(前提是安装了cmake),所以不用在专门的打开cmake的GUI进行编译，而可以直接使用VS2019进行编译。

准备材料

   参考编译说明网址，在windows 下面编译FreeCAD需要三个方面的材料，首先是编译器，这里不用说了，使用的编译器是VS2019, 其次是源代码，参考下载地址，通过命令（预先要安装git）
  git clone https://github.com/FreeCAD/FreeCAD.git
来下载到本地，如果用压缩包的方法下载编译会有一些问题。
下载。第三个就是安装cmake了，这个不在多讲，直接从cmake官网上下载最新的安装文件进行安装即可。最后一个需要的材料是第三方库LibPack参考网址第三方库地址，以上是准备工作，以上内容具备了，我们就可以进行对FreeCAD的编译了。

编译

  在下载完成以上所有材料以后编译需要如下步骤：
  1、我们在vs2019中选择打开下载的FreeCAD的那个文件夹。这时候VS2019会自动集成cmake。我们选择“项目”->“FreeCAD的cmake设置”在里面设置FREECAD_LIBPACK_DIR 为你下载的第三方库后解压的文件夹路径。
  2、选择“项目”->“为FreeCAD的生成缓存”。
  3、生成缓存以后选择“生成”->“全部生成”，此时会出错，如下图所示：

解决办法：在Visual Studio菜单栏上点击“文件”->“高级保存选项”，在“编码”下拉框选择“简体中文（GB2312）”，然后“确定”即可。
  4、重新生成，可能时间会比较长。同时选择启动项目为FreeCAD_d.exe，即可成功。

优势

   1、避免同时使用cmake GUI以及VS进行两线作战。
   2、为fcbt使用提供方面，fcbt生成mod是一个cmake项目，我们生成mod以后，在vs中可以重新生成cmake缓存，从而将mod加载进来。

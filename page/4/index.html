浅谈FreeCAD中的模块

发表于 2022-03-15 分类于 FreeCAD

FreeCAD运行的三种模式

&emsp;&emsp;FreeCAD有三种运行模式，即命令行模式、脚本模式以及界面模式。针对于FreeCAD中每一个模块，都包括两个库，第一个库是App库主要是供脚本调用库，是无界面的功能实现库，第二个库是GUI库，该库是该模块的界面库，界面库是通过调用脚本库来实现功能的。

FreeCAD中的模块加载方法

&emsp;&emsp;主要包含两个大问题：首先是每个模块的加载方法，其次是模块中接口的调用方法。

模块的加载方法

&emsp;&emsp; 加载模块需要遍历每一个模块，而遍历每一一个模块的方法在Application::runInitGuiScript(void)中，这个函数中运行了脚本FreeCADGuiInit.Py，在脚本中会运行函数InitApplications()，这个函数会遍历所有的模块，运行每一个模块的InitGui.py脚本，在该脚本中会运行Gui.addWorkbench(*Workbench())对应的C++代码是Application::sAddWorkbenchHandler，该函数在Gui模块里面进行了封装，对应的函数名就是addWorkbench。最终该WorkBench会被放入到Application的_pcWorkbenchDictionary一个字典列表中。

接口的调用方法

&emsp;&emsp;在python中运行一个导入模块的时候会首先运行Application::activateWorkbench函数，该函数中调用模块下面的InitGui.py文件中平台的Initialize函数，通过以下两句实现：

Initialize函数才是真正加载该模块的方法。

&emsp;&emsp;在Initialize函数中有会调用import 模块名，这会导致调用PyInit_模块名的函数。例如：针对PartGui 模块，Application::activateWorkbench会导致PartWorkbench中的Initialize被调用，进而导致import PartGui被调用，这会直接调用PyInit_PartGui函数。在FreeCAD的GUI模块中，通常该接口实现定义在App&#42;GUI.cpp 文件中（其中 &#42;表示模块名称），例如AppPartGUI.cpp 定义了PyInit_PartGui。

&emsp;&emsp;PyInit_PartGui主要进行了如下的工作：

1、加载脚本模块：import Part

2、创建PartGui对应的python object。

3、初始化PartGui中的类型。

4、初始化Workbench，Workbench主要是创建菜单以及工具栏等操作。

5、创建命令。

6、加载资源。

&emsp;&emsp;PyInit_PartGui最后会创建一个PyObject，也就是说会创建一个python的类。这样PartGui就可以当作python的一个模块，按照python的调用方式进行调用了。

&emsp;&emsp;PPy::ExtensionModule 是C++封装为python工具PyCXX 提供的封装类，PartGui模块继承了Py::ExtensionModule也是就说采用的PyCXX的方式对C++进行封装的。

国与摇篮

发表于 2022-02-15 更新于 2022-03-15 分类于 历史

&emsp;&emsp;我记得从小学上社会课的时候，老师就一直在跟我们讲，中国是四大文明古国，当时觉得为自己的国家感到骄傲。但是随着长大发现，国家这个名字的含义其实并不简单。国字是一个口，一个戈，一个框，口代表的是人口，戈代表的是兵器框代表的是边界，这样国家最起码包含两层含义：首先是地理的概念，有了地理才有国的位置，再次必须有人，有了人才有了国家的灵魂。四大文明古国中的国应该是当时的地理以及当时在那个地方生活的人共同组成的，不过地方是永存的，但是人是流动的，现在一个地方的人和以前这个地方的人也许已经不是同一族群了，四大文明古国应该叫做四大文明摇篮更为合适，因为摇篮指的是一种地域概念，存在一定的不变性，而且具有客观性。

&emsp;&emsp;历史课上，老师讲过距今二十万年前的北京人，八十万年前的陕西蓝田人以及一百五十万年前的云南元谋人，这些人其实和我们现在的人基本没有任何的关系，他们是直立人而我们是智人的后代，彼此之间存在着生成隔离。我们所有人的祖先是距今大约10万年前开始从非洲多次分批迁徙出来的（出非洲时大家应该都是黑色的）。人类在不断的迁徙，每个地域的人在不断的迭代，相互融合才有现代人类的样子。远古文明从中亚向世界扩散，就中国而言，先是扩散到中国北方，进而南下，大体上来说南方比北方更为纯种，无怪乎有人说唐朝的话更接近于粤语，而北方的话更接近于胡语。

FreeCAD中类型的动态创建

发表于 2022-01-05 更新于 2022-01-06 分类于 FreeCAD

&emsp;&emsp;在我们读取数据时，会根据保存的数据的内容创建对应的类，从而正确的读取数据，根据保存数据创建类就是一种类的动态创建。本博客下面主要讲述FreeCAD中类的动态创建过程。

实现动态创建的重要材料

&emsp;&emsp;在FreeCAD的type.cpp文件中，和动态创建非常相关的几个变量或者说类我们先介绍一下：
Type 类：这个类中封装了类的一般操作，比如创建实例，查找其父类等，这个类会和一个具体的类绑定，通过该类创建出对应的类，这个类应该是动态创建的接口类。
TypeData类： 是保存对应的绑定类的具体数据的类，Type 类应该说是对TypeData类操作的一个接口，TypeData中包含一个Type 类型的变量。
Type类中有两个非常重要的静态变量：
map<string,unsigned int> Type::typemap;
vector<TypeData*> Type::typedata;
这两个变量最终保存了注册类的名称以及创建类的函数。
&emsp;&emsp;FreeCAD所有的类型都是继承自BaseClass类，这个类中有一个静态的Type即classTypeId（子类中会重写覆盖），正是通过这个，所有的类型和Type绑定，实现的动态创建的相关操作。
&emsp;&emsp;所有继承自BaseClass类的类都有一个宏 TYPESYSTEM_HEADER();该宏重新声明了以下函数：
static Base::Type getClassTypeId(void);
static void init(void);
static void *create(void);
以及一个变量
static Base::Type classTypeId；
这些变量在基类中就存在，重新声明会覆盖掉以前的函数以及变量。后面我们会看到这几个函数和变量在该类型的动态创建很有关系。在类的实现中我们可以看到这个宏的定义类似如下所示：
TYPESYSTEM_SOURCE(Base::AbortException,Base::Exception)
这个是TYPESYSTEM_HEADER宏声明的实现。
有了以上宏定义的内容，我们就可以实现动态创建了。

&emsp;&emsp;动态创建的一个基本的思路是这样的：针对一个类首先得有一个注册，注册的目的是将一个类名和类型实例创建的函数（static）绑定，同时保存在一个容器里面。下面是具体的创建方式。

&emsp;&emsp;创建了一个TypeData，这个TypeData将类名、该类的父类的Type以及创建方法进行了保存。同时将这个创建的TypeData放入Type::typedata中。另外将类名以及在Type::typedata中的索引保存到Type::typemap中。

用VS2019编译FreeCAD

发表于 2021-08-15 更新于 2023-02-27 分类于 FreeCAD

&emsp;&emsp; 本博客主要讲怎样在VS2019中编译FreeCAD。VS2019集成可cmake(前提是安装了cmake),所以不用在专门的打开cmake的GUI进行编译，而可以直接使用VS2019进行编译。

准备材料

&emsp;&emsp; 参考编译说明网址，在windows 下面编译FreeCAD需要三个方面的材料，首先是编译器，这里不用说了，使用的编译器是VS2019, 其次是源代码，参考下载地址，通过命令（预先要安装git）
&emsp;&emsp;git clone https://github.com/FreeCAD/FreeCAD.git
来下载到本地，如果用压缩包的方法下载编译会有一些问题。
下载。第三个就是安装cmake了，这个不在多讲，直接从cmake官网上下载最新的安装文件进行安装即可。最后一个需要的材料是第三方库LibPack参考网址第三方库地址，以上是准备工作，以上内容具备了，我们就可以进行对FreeCAD的编译了。

编译

&emsp;&emsp;在下载完成以上所有材料以后编译需要如下步骤：
&emsp;&emsp;1、我们在vs2019中选择打开下载的FreeCAD的那个文件夹。这时候VS2019会自动集成cmake。我们选择“项目”->“FreeCAD的cmake设置”在里面设置FREECAD_LIBPACK_DIR 为你下载的第三方库后解压的文件夹路径。
&emsp;&emsp;2、选择“项目”->“为FreeCAD的生成缓存”。
&emsp;&emsp;3、生成缓存以后选择“生成”->“全部生成”，此时会出错，如下图所示：

解决办法：在Visual Studio菜单栏上点击“文件”->“高级保存选项”，在“编码”下拉框选择“简体中文（GB2312）”，然后“确定”即可。
&emsp;&emsp;4、重新生成，可能时间会比较长。同时选择启动项目为FreeCAD_d.exe，即可成功。

优势

&emsp;&emsp; 1、避免同时使用cmake GUI以及VS进行两线作战。
&emsp;&emsp; 2、为fcbt使用提供方面，fcbt生成mod是一个cmake项目，我们生成mod以后，在vs中可以重新生成cmake缓存，从而将mod加载进来。

矩阵与点的变换

&emsp;&emsp;在三维线性空间中，有一组向量基表示为：$e =\left ( e_{1},e_{2},e_{3}\right)$，其中$e_{1}=\begin{pmatrix}
x_{1}
\y_{1}
\z_{1}
\end{pmatrix}$，$e_{2}=\begin{pmatrix}
x_{2}
\y_{2}
\z_{2}
\end{pmatrix}$,$e_{3}=\begin{pmatrix}
x_{3}
\y_{3}
\z_{3}
\end{pmatrix}$。有一个点$P$在该组向量基下的坐标为$P=\left ( x,y,z \right )$。
&emsp;&emsp; 在该空间中另有一组向量基表示为：$e^{‘}=\left ( e_{1}^{‘},e_{2}^{‘},e_{3}^{‘}\right )$其中$e_{1}^{‘}=\begin{pmatrix}
x_{1}^{‘}
\y_{1}^{‘}
\z_{1}^{‘}
\end{pmatrix}$,$e_{2}^{‘}=\begin{pmatrix} x_{2}^{‘}
\y_{2}^{‘}
\z_{2}^{‘}
\end{pmatrix}$,$e_{3}^{‘}=\begin{pmatrix}
x_{3}^{‘}
\y_{3}^{‘}
\z_{3}^{‘}
\end{pmatrix}$在该向量基下对应的点$P$的坐标$P^{‘}=\left (x^{‘},y^{‘},z^{‘} \right)$那么我们可以得到：
$$
P = P^{‘}e^{‘}e^{-1}, P^{‘} = Pee^{‘-1}
$$
也就是说：
$$
P=\left ( x,y,z \right )=\left (x^{‘},y^{‘},z^{‘} \right)
\begin{pmatrix}
x_{1}^{‘}&x_{2}^{‘}&x_{3}^{‘}
\y_{1}^{‘}&y_{2}^{‘}&y_{3}^{‘}
\z_{1}^{‘}&z_{2}^{‘}&z_{3}^{‘}
\end{pmatrix}*
\begin{pmatrix}
x_{1}&x_{2}&x_{3}
\y_{1}&y_{2}&y_{3}
\z_{1}&z_{2}&z_{3}
\end{pmatrix}^{-1}
$$
$$
P^{‘}=\left (x^{‘},y^{‘},z^{‘} \right)=\left ( x,y,z \right )*
\begin{pmatrix}
x_{1}&x_{2}&x_{3}
\y_{1}&y_{2}&y_{3}
\z_{1}&z_{2}&z_{3}
\end{pmatrix}*
\begin{pmatrix}
x_{1}^{‘}&x_{2}^{‘}&x_{3}^{‘}
\y_{1}^{‘}&y_{2}^{‘}&y_{3}^{‘}
\z_{1}^{‘}&z_{2}^{‘}&z_{3}^{‘}
\end{pmatrix}^{-1}
$$

旋转矩阵的公式

&emsp;&emsp; 设一个向量$v=\left (v_{x},v_{y},v_{z}\right)$在右手坐标系中绕一个轴$r=\left (r_{x},r_{y},r_{z} \right)$ 逆时针旋转$\phi$度（从轴的正方向往负方向看），其中$r$是一个单位向量。那么对应的旋转矩阵是：
$$
R=
\begin{pmatrix}
cos\phi+(1-cos\phi)*r_{x}^{2}&(1-cos\phi)*r_{x}*r_{y}-r_{z}sin\phi&(1-cos\phi)*r_{x}*r_{z}+r_{y}sin\phi
\(1-cos\phi)*r_{x}*r_{y}+r_{z}sin\phi&cos\phi+(1-cos\phi)*r_{y}^{2}&(1-cos\phi)*r_{y}*r_{z}-r_{x}sin\phi
\(1-cos\phi)*r_{x}*r_{z}-r_{y}sin\phi&(1-cos\phi)*r_{y}*r_{z}+r_{x}sin\phi&cos\phi+(1-cos\phi)*r_{z}^{2}
\end{pmatrix}
$$

四元数的基本运算

定义

  • 一个四元数可以进行如下定义：
    $$\hat{q} =(q_{v}, q_{w}) = iq_{x} + jq_{y} + kq_{z} + q_{w} = q_{v} + q_{w}$$
    $$ i^{2} = j^{2} = k^{2} = -1, jk = -kj = i, ki = -ik = j, ij = -ji = k.$$
  • 乘法
    $$ \hat{q}\hat{r} = (q_{v}\times r_{v}+ r_{w}q_{v}+q_{w}r_{v},q_{w}r_{w}-q_{v}\cdot r_{v})$$
  • 共轭
    $$\hat{q}^{*} = (-q_{v}, q_{w})$$
  • 加法
    $$\hat{q} + \hat{v} = (q_{v}+r_{v}, q_{w}+r_{w})$$
  • 规范化
    $$n(\hat{q}) = \sqrt{\hat{q}\hat{q}^{*}} = \sqrt{q_{v}\cdot q_{v} + q^{2}{w}}
    = \sqrt{q    {x}^{2}+ q_{y}^{2}+q_{z}^{2}+q_{w}^{2}}.$$
  • 单位化
    $$\hat{i} = (0,1)$$
  • 逆
    $$\hat{q}^{-1} = \frac{1}{n(\hat{q})^{2}}\hat{q}^{*}$$

四元数与旋转

&emsp;&emsp;设一个向量(或者点)$\hat{p}=\left (p_{x},p_{y},p_{z},p_{w}\right)$,我们设其为一个四元组，假设我们还有一个单位四元组$\hat{q}=\left (sin\phi*u_{q},cos\phi\right)$,那么我们可以得到：
$$ \hat{q}\hat{p}\hat{q}^{-1}$$
表示$P$绕着旋转轴$u_{q}$旋转$2\phi$得到的结果，其中$u_{q}$是一个单位向量。

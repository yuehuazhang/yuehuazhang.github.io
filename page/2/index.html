Vulkan笔记-API安装

发表于 2022-10-15 分类于 Vulkan

   这是Vulkan学习的第一节，怎样安装VulkanAPI，由于我使用的是Windows,我这里只讲述Windows的安装。

需要库介绍

Vulkan SDK

  • 安装： 我们可以从LunarG网站上面下载Vulkan的SDK，安装方法也是傻瓜式的安装。
  • 判断当前电脑是否支持Vulkan:在SDK的安装目录下面运行一下vkcube.exe这个demo，如果可以正常运行那么表示当前显卡支持Vulkan。
  • 安装目录下有用的工具：glslangValidator.exe以及 glslc.exe，这两个工具分别是用于编译顶点着色器以及片元着色器为字节码，后续使用时再详细介绍。

    GLFW

    Vulkan与窗口无关，所以需要一个窗口创建的库。GLFW可以用于创建Window界面。

    GLM

    GLM是一个矩阵操作库，他是一个模板库，也就是说其只有头文件。

    创建开发工程

  • 使用VS创建一个Console Application，并添加一个CPP文件。
  • 设置include目录：C++ -> General -> Additional Include Directories 将Vulkan、GLFW和GLM的头文件目录已经其他需要引用的头文件路径添加进去。
  • 添加附加包含目录：需要使用的库所在的目录。
  • 添加附加依赖库，包括：glfW3.lib（静态库文件）、vulkan-1.lib。

Blender中的快捷键学习

发表于 2022-09-24 更新于 2022-09-25 分类于 Blender

Blender中的快捷键学习

平移

  • 选中物体，按住G键移动鼠标进行物体平移。
  • 选中物体，按住G键，然后按X/Y/Z键，移动鼠标，物体会沿着X/Y/Z轴进行平移。
  • 选中物体，按住G键，然后按X/Y/Z键，然后输入数值（正负都可）物体会沿着X/Y/Z轴平移相应的距离。

    缩放

  • 选中物体，按住S键，实现物体缩放。
  • 选中物体，按住S键，然后按X/Y/Z键，移动鼠标，物体会沿着X/Y/Z轴进行缩放。
  • 选中物体，按住S键，然后按X/Y/Z键，然后输入数值（正负都可）物体会沿着X/Y/Z轴缩放相应的大小。
  • 在编辑物体的模式下Atrl+S 进行法向缩放。

    游标

  • 游标定义：游标是新建几何体的中心。
  • 按住Shift+C键 游标回复到坐标中心。
  • 按住Shift同时点击鼠标右键，游标移动到鼠标点击位置。

    选择以及删除

  • A键全选。
  • X键删除选中的物体。

    切换

  • tab键切换物体环境和编辑环境。
  • 按Z键，切换显示模式（线框、实体、材质预览、渲染预览等）。
  • 编辑环境下：可以拾取点、线、面，切换的快捷键是1（点），2（线），3（面）。
  • Alt+Z切换透视模式。透视模式下容易对背面进行选择。
  • 几个视图的切换：小键盘1是前视图（Ctrl+1反向），小键盘3是右视图（Ctrl+1反向），小键盘7是顶视图（Ctrl+1反向），小键盘9是底视图（Ctrl+1反向）。
  • 按~键也可以切换视图。

    创建以及编辑几何体

  • Shift+A 创建几何体。
  • 在编辑模式下按E挤出。
  • Ctrl+R 循环切割，按鼠标左键移动切换的位置，按鼠标左键后点右键切割到中心，点击鼠标滚轮可以产生多条切割线。
  • 选中面，点击键盘i键，产生内插面。

FreeCAD运行的三种模式

   FreeCAD有三种运行模式，即命令行模式、脚本模式以及界面模式。针对于FreeCAD中每一个模块，都包括两个库，第一个库是App库主要是供脚本调用库，是无界面的功能实现库，第二个库是GUI库，该库是该模块的界面库，界面库是通过调用脚本库来实现功能的。

FreeCAD中的模块加载方法

每个模块的加载方法。

  FreeCAD启动的时候会遍历安装目录mod文件夹下面的文件，该遍历的实现位于Application::runInitGuiScript(void)中。
这个函数中运行了脚本FreeCADGuiInit.Py，在脚本中会运行函数InitApplications()，这个函数会遍历所有的模块，运行每一个模块的InitGui.py脚本，在该脚本中会运行Gui.addWorkbench(*Workbench())对应的C++代码是Application::sAddWorkbenchHandler，该函数在Gui模块里面进行了封装，对应的函数名就是addWorkbench。最终该WorkBench会被放入到Application的_pcWorkbenchDictionary一个字典列表中。

   在python中运行一个导入模块的时候会首先运行Application::activateWorkbench函数，该函数中调用模块下面的InitGui.py文件中平台的Initialize函数，通过以下两句实现：


Initialize函数才是真正加载该模块的方法。
   在Initialize函数中有会调用import 模块名，这会导致调用PyInit_模块名的函数。例如：针对PartGui模块，Application::activateWorkbench会导致PartWorkbench中的Initialize被调用，进而导致import PartGui被调用，这会直接调用PyInit_PartGui函数。在FreeCAD的GUI模块中，通常该接口实现定义在AppGUI.cpp 文件中（其中表示模块名称），例如AppPartGUI.cpp 定义了PyInit_PartGui。
PyInit_PartGui主要进行了如下的工作：

  • 加载脚本模块：import Part
  • 创建PartGui对应的python object。
  • 初始化PartGui中的类型。
  • 初始化Workbench，Workbench主要是创建菜单以及工具栏等操作。
  • 创建命令。
  • 加载资源。
       PyInit_PartGui最后会创建一个PyObject，也就是说会创建一个python的类。这样PartGui就可以当作python的一个模块，按照python的调用方式进行调用了。
       Py::ExtensionModule 是C++封装为python工具PyCXX 提供的封装类，PartGui模块继承了Py::ExtensionModule也是就说采用的PyCXX的方式对C++进行封装的。

国与摇篮

发表于 2022-02-15 更新于 2022-03-15 分类于 历史

  我记得从小学上社会课的时候，老师就一直在跟我们讲，中国是四大文明古国，当时觉得为自己的国家感到骄傲。但是随着长大发现，国家这个名字的含义其实并不简单。国字是一个口，一个戈，一个框，口代表的是人口，戈代表的是兵器框代表的是边界，这样国家最起码包含两层含义：首先是地理的概念，有了地理才有国的位置，再次必须有人，有了人才有了国家的灵魂。四大文明古国中的国应该是当时的地理以及当时在那个地方生活的人共同组成的，不过地方是永存的，但是人是流动的，现在一个地方的人和以前这个地方的人也许已经不是同一族群了，四大文明古国应该叫做四大文明摇篮更为合适，因为摇篮指的是一种地域概念，存在一定的不变性，而且具有客观性。

  历史课上，老师讲过距今二十万年前的北京人，八十万年前的陕西蓝田人以及一百五十万年前的云南元谋人，这些人其实和我们现在的人基本没有任何的关系，他们是直立人而我们是智人的后代，彼此之间存在着生成隔离。我们所有人的祖先是距今大约10万年前开始从非洲多次分批迁徙出来的（出非洲时大家应该都是黑色的）。人类在不断的迁徙，每个地域的人在不断的迭代，相互融合才有现代人类的样子。远古文明从中亚向世界扩散，就中国而言，先是扩散到中国北方，进而南下，大体上来说南方比北方更为纯种，无怪乎有人说唐朝的话更接近于粤语，而北方的话更接近于胡语。

FreeCAD中类型的动态创建

发表于 2022-01-05 更新于 2022-01-06 分类于 FreeCAD

  在我们读取数据时，会根据保存的数据的内容创建对应的类，从而正确的读取数据，根据保存数据创建类就是一种类的动态创建。本博客下面主要讲述FreeCAD中类的动态创建过程。

实现动态创建的重要材料

  在FreeCAD的type.cpp文件中，和动态创建非常相关的几个变量或者说类我们先介绍一下：
Type 类：这个类中封装了类的一般操作，比如创建实例，查找其父类等，这个类会和一个具体的类绑定，通过该类创建出对应的类，这个类应该是动态创建的接口类。
TypeData类： 是保存对应的绑定类的具体数据的类，Type 类应该说是对TypeData类操作的一个接口，TypeData中包含一个Type 类型的变量。
Type类中有两个非常重要的静态变量：
map Type::typemap;
vector Type::typedata;
这两个变量最终保存了注册类的名称以及创建类的函数。
  FreeCAD所有的类型都是继承自BaseClass类，这个类中有一个静态的Type即classTypeId（子类中会重写覆盖），正是通过这个，所有的类型和Type绑定，实现的动态创建的相关操作。
  所有继承自BaseClass类的类都有一个宏 TYPESYSTEM_HEADER();该宏重新声明了以下函数：
static Base::Type getClassTypeId(void);
static void init(void);
static void *create(void);
以及一个变量
static Base::Type classTypeId；
这些变量在基类中就存在，重新声明会覆盖掉以前的函数以及变量。后面我们会看到这几个函数和变量在该类型的动态创建很有关系。在类的实现中我们可以看到这个宏的定义类似如下所示：
TYPESYSTEM_SOURCE(Base::AbortException,Base::Exception)
这个是TYPESYSTEM_HEADER宏声明的实现。
有了以上宏定义的内容，我们就可以实现动态创建了。

动态创建的实现

  动态创建的一个基本的思路是这样的：针对一个类首先得有一个注册，注册的目的是将一个类名和类型实例创建的函数（static）绑定，同时保存在一个容器里面。下面是具体的创建方式。
  我们注册一个变量的时候会调用其init函数这个函数的实现在TYPESYSTEM_SOURCE中，这个函数主要进行了以下的操作：
  创建了一个TypeData，这个TypeData将类名、该类的父类的Type以及创建方法进行了保存。同时将这个创建的TypeData放入Type::typedata中。另外将类名以及在Type::typedata中的索引保存到Type::typemap中。
在实际读取数据中，具体过程是这样的：
当读到一个类名的时候，会调用Type::createInstanceByName 函数，在这个函数中，会根据读取的类的名称在Type::typemap找到对应的索引，通过对应的索引在Type::typedata中找到对应的TypeData，TypeData中保存该类的创建函数，那么我们调用创建函数就可以创建出该类来。
